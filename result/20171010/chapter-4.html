著者について

ヘイドン・ピカリング（Heydon Pickering）は実用デザイナーであり、ライターであり、講演も行っています。また「Smashing Magazine」（脚注1）のアクセシビリティ担当エディターで、The Paciello Group（脚注2）のコンサルタントも務めています。Webをインクルーシブな場所にするための、新しい方法や革新的な手段に関心を持っています。そのために、ユーザー調査、システム思考、プレーンで昔ながらのセマンティックを持つHTMLといったものすべてを活用しています。文章を書いたり、コードを書いたり、イラストを描いているとき以外は、メンタルヘルスに関するキャンペーンを行ったり、サウンドデザインに挑戦したり、自分でデチューンしたSGのコピーギターでドゥームメタルのリフを繰り返し弾いたりしています。そんな彼の燃料の大半は、オリジナルレシピのギネスとナガチリです。

レビュアーについて

ロドニー・レーム（Rodney Rehm）はドイツ南部で活動しているWeb開発者です。10年間、フリーランスのフルスタックエンジニアとして働いた後、ドイツテレコムのスマートホームプラットフォームQivicon（キビコン）（脚注3）のフロントエンドエンジニアになりました。URI.js（脚注3）とアクセシビリティライブラリally.js（脚注4）を作ったり、ブラウザ動くlibsass（脚注5）を作ったり、世界初のbuggyfill（脚注6）を書いたりしています。

スティーブ・フォルクナー（Steve Faulkner）は、TPG EuropeのシニアWebアクセシビリティコンサルタント兼テクニカルディレクターです。2006年にPaciello Group（脚注8）に入り、Vision Australia（脚注7）ではシニアWebアクセシビリティコンサルタントも務めました。アクセシビリティテストツールWeb Accessibility Toolbar（脚注8）の考案者であり、リードデベロッパーでもあります。スティーブは、W3C Web Platformsワーキンググループ、W3C ARIAワーキンググループなど、いくつかのグループのメンバーとなっています。W3C（脚注9）でさまざまな仕様の策定に携わっており、HTML 5.1（脚注10）、ARIA in HTML（脚注11）、Notes on Using ARIA in HTML（脚注12）、 HTML5: Techniques for useful text alternatives（脚注13）などを手掛けています。また、HTML5accessibility.com（脚注16）の開発と運用も手掛けています。
